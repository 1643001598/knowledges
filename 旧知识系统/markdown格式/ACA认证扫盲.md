  - 1. 1 从人工智能到大模型的演变
    - 人工智能
      - def
        - 人工智能（AI）是一门使机器模拟人类智能过程的学科，其中具体包括学习、推理、自我修正、感知和处理语言等功能。
      - ftr
        - 人工智能涉及计算机科学、数学、心理学等众多领域的知识，通过创建能够实现智能行为的算法或软件系统，来表现出与人类的智能行为相似的特性。
      - 子领域
        - 机器学习
          - 子领域
            - 按学习范式分类
              - 监督学习
                - def
                  - 数据有“标准答案”（标签）。主要用于分类（如猫狗识别）和回归（如房价预测）。
              - 无监督学习
                - def
                  - 数据没有标签。主要用于发现内在结构（如客户分群、数据降维）。
              - 强化学习
                - def
                  - 通过与环境互动、获得奖励或惩罚来学习策略。主要用于游戏AI、机器人控制等。
              - 半监督学习
            - 按模型/技术分类
              - 决策树
              - 支持向量机
              - 线性/逻辑回归
              - 深度学习
                - 子领域
                  - 深度神经网络
                  - CNN(卷积神经网络)
                  - RNN/LSTM/Transformer
                  - 生成对抗网络
                - 应用方向
                  - 生成式人工智能
                    - def
                      - 一类能够根据学习到的数据分布，生成全新的、原创内容的人工智能模型和技术
                    - 子领域
                      - 大语言模型
                      - 扩散模型
                      - 生成对抗网络
                      - 变分自编码器
                - def
                  - 使用包含多个隐藏层的人工神经网络（即“深度神经网络”）来自动学习数据的层次化特征表示。
          - def
            - 核心思想是让计算机系统利用数据而非显式编程指令来学习和改进。
          - content
            - 数据：学习的原材料（文本、图像、数值等）。
            - 模型：用来从数据中学习模式的数学结构（如神经网络、决策树）。
            - 学习过程：通过优化算法不断调整模型参数，使模型在已知数据上的表现越来越好。
  - 1.2 大模型，它来了
    - 历史
      - 2021年
        - 斯坦福大学的研究员团队发表了一篇论文，提出了Foundational Models（基础模型，即大模型）的概念。简单来说，它是一类具有大量参数（通常在十亿以上），能在极为广泛的数据上进行训练，并适用于多种任务和应用的预训练深度学习模型。
      - 2022年
        - 11月
          - OpenAI公司发布了ChatGPT——一种先进的人工智能语言模型，专为对话交互而设计，具有强大的自然语言理解和生成能力，可以完成撰写论文、邮件、脚本、文案、翻译、代码等任务。ChatGPT的发布标志着AI大模型在语言理解与生成能力上的重大突破，对全球AI产业产生了深远影响，开启了人工智能大模型应用的新篇章。
      - 2023年
        - 3月
          - 国内厂商纷纷发布各自研发的大语言模型产品。百度发布文心一言、阿里巴巴集团千问大模型开始企业内测、商汤科技带来了商量SenseChat、360启动360智脑企业内测、华为盘古大模型、科大讯飞1+N认知大模型、昆仑万维的天工大模型、以及京东言犀、腾讯混元......百模大战一触即发。
        - 8月
          - 阿里巴巴集团发布了千问系列开源大模型，并相继推出了7B（约70亿参数）、72B（约720亿参数）等不同参数规模的大语言模型版本。目前，千问系列大语言模型已升级至Qwen3版本，具备自然语言理解、文本生成、视觉理解、音频理解、工具使用、角色扮演、作为AI Agent进行互动等多种能力。

## 画布 2: AI

# AI核心观念扫盲

- 核心观念
  - MCP
    - 核心认知：为什么认识MCP
      - 核心问题
        - 在 MCP 出现之前，让 AI（如 Claude、GPT）调用一个外部工具（比如查天气、操作数据库）会非常麻烦。你需要为每一个模型单独写一套集成代码，这被称为“碎片化”问题
      - 解决方式
        - 传统方式：AI模型A → 自定义代码 → 天气API；AI模型B → 另一套代码 → 天气API。每次更换或新增模型，都要重复劳动。
        - MCP方式：AI模型 (MCP客户端) → 统一的 MCP 协议 → 天气 MCP 服务器 → 天气API。一次开发，处处可用
    - 核心架构：三大角色
      - 角色
        - MCP 主机 (Host)
          - 你直接交互的 AI 应用程序。比如 Claude Desktop、Cursor IDE、或你正在使用的各种 AI 编程工具。它是发起请求的“大脑”
        - MCP 客户端 (Client)
          - 它内嵌在主机中，负责与 MCP 服务器建立一对一的连接，是主机与服务器之间的“通信官”
        - MCP 服务器 (Server)
          - 轻量级的程序，它暴露特定功能。比如 GitHub 服务器（帮你管理仓库和PR）、PostgreSQL 服务器（让你用自然语言查询数据库）。每个服务器都专注于一件事
      - 如何写协作
        - 当你向主机（如 Claude Desktop）提问：“帮我看看 GitHub 上最近的 issue”。主机内部的客户端会将这个请求通过标准格式发给 GitHub MCP 服务器。服务器去 GitHub 获取数据，然后将结果原路返回。主机再将结果呈现给你
    - 关键特性与术语
      - 工具 (Tools)
        - 服务器暴露的具体功能，比如“创建一个 issue”、“查询天气”。每个工具都有明确的名称、描述和输入参数
      - 资源 (Resources)
        - 服务器能访问的数据，比如文件内容、数据库记录
      - 传输方式
        - stdio (标准输入/输出)
          - 最常用的一种方式。MCP 服务器作为本地的一个子进程运行。你在配置时看到的 command 和 args 字段（如 "command": "npx"）就是在定义如何启动这个本地进程
            - 进程分离模型
        - HTTP/SSE (服务器发送事件)
          - 用于连接远程的 MCP 服务器，支持更复杂的认证场景
    - 学习路径
      - 第一步：理解“为什么” (1天)
        - 目标：深刻理解 MCP 要解决的核心痛点。
        - 行动：阅读并对比传统 API 与 MCP 在开发成本、交互模式上的差异。
      - 第二步：动手“跑起来” (2-3天)
        - 目标：亲手配置并运行一个 MCP 服务器，获得直观感受。
        - 行动
          - 选择一个支持 MCP 的客户端。最推荐的是 Claude Desktop。
          - 从最简单的服务器开始，比如文件系统服务器，让 Claude 能读取你电脑上的文件。
          - 按照教程，完成一个GitHub MCP 服务器的安装和配置。这是最常用也最能体现MCP价值的实战。
          - 试着用自然语言对 Claude 说：“列出我的GitHub仓库里最近的3个issue”，看看效果。
      - 第三步：开始“自己造” (进阶)
        - 目标：理解如何开发自己的 MCP 服务器。
        - 行动：
          - 学习官方推荐的 Python (FastMCP) 或 Node.js 框架。
          - 从一个极简的例子开始，比如“一个计算器服务器”，它只暴露“加法”和“减法”两个工具。
          - 尝试将它集成到你配置好的 Claude Desktop 中，并成功调用。
